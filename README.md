## Featured Projects

### [`mcp-homelab`](https://github.com/asvarnon/mcp-homelab) — AI-powered homelab management

An [MCP](https://modelcontextprotocol.io/) server that gives AI assistants (Claude, Copilot, etc.) real-time access to homelab infrastructure. 23 tools across SSH, Proxmox VE, OPNsense, and Docker — config-driven, typed returns, no hardcoded hosts.

**Stack:** Python · FastMCP · Paramiko · httpx · Pydantic v2 · pytest  
**Published on [PyPI](https://pypi.org/project/mcp-homelab/)** · `pip install mcp-homelab`

### [`connections`](https://github.com/asvarnon/connections) — Discord bot

A Spring Boot Discord bot using JDA for guild management. Dynamic command system for querying players and artisans with static and criteria-based lookups, backed by PL/SQL.

**Stack:** Java · Spring Boot · JDA · PL/SQL

### [`data_pipeline_av`](https://github.com/asvarnon/data_pipeline_av) — Streaming data pipeline

Learning project: streaming ingestion → validation → Parquet → DuckDB, with dbt-core style transforms and a Grafana dashboard.

**Stack:** Python · DuckDB · Parquet · Grafana

### [`kali_linux_tools`](https://github.com/asvarnon/kali_linux_tools) — Security tooling

Tools, practice scripts, and experiments for Kali Linux. Exploring offensive security and penetration testing fundamentals.

**Stack:** Python · Kali Linux

## Philosophy

I build tools to solve problems I actually have, and try to avoid hypothetical ones. If I can make something generic enough to share, I publish it. The goal is always to learn by shipping — not by collecting tutorials.

Right now I'm focused on AI-augmented infrastructure tooling: giving AI assistants real access to real systems, so the conversation stops being "here's my `docker ps` output" and starts being "fix my container."
